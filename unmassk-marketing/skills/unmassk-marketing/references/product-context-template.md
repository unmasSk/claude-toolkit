# Product Marketing Context Template

Create a `product-marketing-context.md` file for each project before running any unmassk-marketing skill. All marketing skills read this file to avoid repeating foundational positioning and messaging information.

Store the completed file at `.agents/product-marketing-context.md` in the project root.

Fill every section below. Skip sections that do not apply (e.g., Personas for B2C products). Use verbatim customer language wherever possible -- exact phrases outperform polished descriptions.

---

## 1. Product Overview

- **One-liner:** Single sentence describing the product.
- **What it does:** 2-3 sentences on core functionality.
- **Product category:** The "shelf" customers search for you on.
- **Product type:** SaaS, marketplace, e-commerce, service, etc.
- **Business model:** Pricing structure and revenue model.

## 2. Target Audience

- **Target companies:** Industry, size, stage.
- **Decision-makers:** Roles and departments involved.
- **Primary use case:** The main problem solved.
- **Jobs to be done:** 2-3 things customers "hire" the product for.
- **Use cases:** Specific scenarios where the product applies.

## 4. Problems and Pain Points

- **Core problem:** The challenge customers face before finding you.
- **Why alternatives fall short:** What current solutions miss.
- **What it costs them:** Time, money, opportunities lost.
- **Emotional tension:** Stress, fear, doubt customers experience.

## 5. Competitive Landscape

- **Direct competitors:** Same solution, same problem. State how each falls short.
- **Secondary competitors:** Different solution, same problem. State how each falls short.
- **Indirect competitors:** Conflicting approach entirely. State how each falls short.

## 6. Differentiation

- **Key differentiators:** Capabilities alternatives lack.
- **How we do it differently:** The distinct approach or mechanism.
- **Why that is better:** Concrete benefits of the approach.
- **Why customers choose us:** The decisive factor in switching.

## 8. Switching Dynamics

Apply the JTBD Four Forces framework:

- **Push:** Frustrations driving customers away from their current solution.
- **Pull:** What attracts them to this product.
- **Habit:** What keeps them stuck with their current approach.
- **Anxiety:** What worries them about switching.
